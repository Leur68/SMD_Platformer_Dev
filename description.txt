1. 

У консоли есть Color RAM (оперативная память цветов). Она состоит из четырех палитр по 16 цветов каждая. 
В единицу времени (один кадр) могут использоваться только 64 цвета, находящиеся в этих палитрах в CRAM.
Небольшой нюанс: у каждой палитры первый цвет - прозрачный, поэтому можно считать что всего цветов в CRAM 60.
Всего же (не в одном кадре, а в теории) консоль поддерживает 512 цветов.
То есть цвета для игры слелует выбирать из специальной палитры, состоящей из 512 цветов (я ее скину).
Но в течение игры CRAM может сколько угодно редактироваться.

2. 

Игра состоит из трех основных слоев. Назовем их так: фон, земля и спрайты.

Фон это то с чем игрок не взаимодействует, но он прокручивается при движении. Он находится позади остальных объектов.
Можно прокручивать горизонтальными строками с разной скоростью для создания эффекта паралакса (глубины), например облака прокручивать с одной скоростью, а деревья с другой. 
Это делается на уровне программирования, нужно только нарисовать объекты в определенных горизонтальных областях.

Земля это то с чем игрок может взаимодействовать, но оно неподвижно. Только прокручивается при движении игрока по игровому миру. Земля, поверхности, лесенки, препятствия и т.д. 

Фон и земля не анимированные.

Можно сымитировать анимацию фона и земли путем подмены цветов в палитре. Например чтобы сделать эффект течения воды, как сделано в игре Sonic (можешь посмотреть прохождение в интернете). 
Цвета в палитре, которую использует слой, подменяются таким образом, что создается эффект течения воды. 
Нужно только подобрать цвета, сам эффект делается в коде.

Спрайты это наиболее интересный слой. Каждый спрайт - отдельный объект. Хотя спрайтов много, у каждого спрайта может быть своя палитра (одна из четырех имеющихся в CRAM). 
Например для персонажа можно завести совершенно отдельную палитру, чтобы сделать его максимально красивым. 
А для двигающейся платформы использовать ту же палитру, которая отведена для слоя земли.
Ведь слой земли не может быть анимированным, а спрайты могут быть и могут как угодно перемещаться по экрану. Поэтому двигающуюся платформу придется делать спрайтом.
Спрайты анимируются с помощью обычных картинок по определенному шаблону (в разных областях располагаются разные кадры), я скину пример.

3. 

Цвета в любой из 4 палитр могут подменяться программно в любой момент. 
Тогда у всех объектов (будь то спрайты или фон), которые опираются на эту палитру, эти цвета будут сразу меняться.
Это можно использовать чтобы сделать игру более красивой.
Можно даже все палитры залить одним цветом, так делается чтобы например создать эффект грозы, вспышки или полной темноты.
Или затемнить персонажа в более темной локации, программно сделав цвета более тусклыми.

4. 

Можно использовать хитрость чтобы сделать персонажа еще более красивым - например сделать его состоящим из двух спрайтов, каждый из которых будет иметь свою палитру. 
Тогда количество цветов у персонажа будет в 2 раза больше. Они будут двигаться синхронно, но не накладываясь друг на друга. Это необязательно.
Если рисовать заставку для меню или титров (когда надо просто на весь экран показать большую картинку), 
можно разбить её на 4 части, которые будут наложены друг на друга и в каждой из которых будут использоваться 15 уникальных цветов в одной из палитр.
Тогда результат будет выглядеть необычно красиво для этой консоли. 
Если это будет не сложно сделать, то я буду рад. Заставку для меню.

5. 

Определенные цвета в CRAM выбираются для каждого экрана в игре. То есть, для меню можно подобрать одни цвета, а для самой игры уже совсем другие, какие понадобятся.
Также использование цветов можно менять в зависимости от текущего места на игровой карте. 
Например, игровая карта шириной в 2 экрана. На обоих половинах есть анимированные спрайты-цветочки. 
Цветочки используют например палитру 3, отведенную для спрайтов.
Но на левой половине цветочки будут использовать одни 16 цветов, а на правой совершенно другие. Содержимое CRAM будет меняться при прокручивании игрового мира.
Важно только, что в одном кадре на экране могут отображаться только 64 цвета (вместе с 4 прозрачными).

6.

Изображение состоит из тайлов - это маленькое изображение 8 на 8 пикселей. Это минимально возможное изображение, которое может выводить консоль. 
Важно, что все изображения по размерам должны быть кратны восьми. 
Экран размером 320 (ширина) на 224 (высота).
Персонаж размером 24 на 24.

7. 

Все изображения должны быть в индексированном формате с расширением png и содержать только те цвета, которые должны быть в отведенной для них палитре в CRAM. 
И если два изображения используют одну палитру, цвета должны быть в одном и том же порядке. 
Даже если конкретное изображение не использует какой-то цвет из палитры, он все равно там должен быть.
Одно изображение привязывается к одной палитре и пользуется ее цветами.
Первый цвет в палитре всегда прозрачный.
